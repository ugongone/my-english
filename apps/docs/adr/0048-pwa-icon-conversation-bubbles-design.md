# 0048 – PWAアイコンを「2つの吹き出し」デザインに全面刷新

## 背景 / Context

- 0047で行ったアイコン刷新は、従来の「チャット吹き出し＋マイク」というコンセプトを踏襲した改善だった。
- その後、ユーザーから「前のデザインは一切参考にしなくていい。チャット／会話／AIというテーマから、要素を変えていくつか試作してほしい」という要望があり、既存コンセプトに縛られない5案（会話＝2つの吹き出し、AI＝スパークル、チャット＝入力中ドット、AI＝ネットワークノード、会話＝ダブルクォーテーション）を新規に作成し、比較用ページで提示した。
- 比較の結果、ユーザーは「会話＝2つの吹き出し」案を採用すると回答した。

---

## 決定 / Decision

`public/icon.svg` を「奥と手前に重なる2つの吹き出し」で会話を表現するデザインに全面的に置き換え、`icon-192x192.png` / `icon-512x512.png` / `apple-touch-icon.png` を再生成する。マイク・音声波形・リフレッシュバッジなど旧デザインの要素は完全に廃止する。

---

## 理由 / Rationale

- マイクや音声波形は「音声入力」を強く連想させるが、アプリの本質は「AIとの会話（テキスト・音声どちらも含む）」であり、2人の吹き出しが重なる構図の方がプロダクトの核（会話）をシンプルに表現できる。
- 単一の吹き出しではなく2枚を奥行きをつけて重ねることで、「一方向の発話」ではなく「双方向の会話」であることを視覚的に示せる。
- 要素を削ぎ落としたことで、192px/180pxのような小サイズでも図形の輪郭が潰れず視認性が高い。

---

## 実装詳細 / Implementation Notes

### 1. アイコンSVGを2吹き出しデザインに置き換え

```svg
<!-- apps/web/public/icon.svg -->
<rect width="512" height="512" rx="112" fill="url(#bg)"/>

<!-- 奥の吹き出し（会話の相手） -->
<rect x="82" y="92" width="272" height="188" rx="52" fill="white" opacity="0.45"/>
<path d="M118 272 L86 328 L168 276 Z" fill="white" opacity="0.45"/>

<!-- 手前の吹き出し（自分） -->
<rect x="164" y="232" width="272" height="188" rx="52" fill="white"/>
<path d="M400 404 L434 456 L354 408 Z" fill="white"/>
```

理由:

- 奥の吹き出しを半透明（`opacity="0.45"`）にすることで、手前の吹き出しとの前後関係＝奥行きを表現。
- 背景は0047から踏襲したインディゴ系グラデーション（`#818CF8` → `#4338CA`）ではなく、会話案検討時に新たに定義したグラデーションをそのまま採用し、旧デザインの配色から明確に差別化した。

### 2. PNGアイコンの再生成

0047と同じく `icon.svg` を単一ソースとして sharp で192/512/180の3サイズを機械生成した（生成スクリプト自体に変更なし）。

---

## 影響 / Consequences

- ホーム画面・タスク切替時に表示されるPWAアイコンの見た目が、マイクを想起させるデザインから「吹き出しが重なる会話」デザインに変わる。
- `manifest.json` の構成（ファイル名・サイズ・`purpose`）は変更していないため、マニフェスト側の追随対応は不要。
- 0047時点で未対応だった `screenshot-wide.png` / `screenshot-narrow.png` の欠落、iOSスプラッシュ画面（`apple-touch-startup-image`）の未設定は、引き続き本対応の範囲外。
